## O18 — Event-driven coverage invalidation — **BUILD**
**Gap (verified):** coverage refresh triggers only on a new reported quarter; the $84.75B raise (vs stale $80B) never flagged coverage. **Design:** add an event-trigger set to the refresh path — when a material event postdates the coverage `as_of` (major equity/debt financing, mandatory convertible, acquisition, new capex guidance, material legal ruling, restructuring, segment redef, dividend/buyback change, guidance withdrawal), emit a `coverage_stale: {reason, event_date}` flag and surface it in QC + the report footer. Detection is deterministic from the snapshot's `events`/`catalysts` + a `coverage.as_of`; no LLM judgment, no calibration constant. **Acceptance:** a GOOG-like bundle whose coverage predates a financing catalyst flags `coverage_stale`; a current bundle does not. **Files:** `refresh_plan.py`/`coverage_qc.py` + a `report_qc` disclosure check.

## O19 — Risk-unit sizing output + entry-state field — **BUILD**
**Design (review-prescribed, Portfolio-OS handoff):** add a **risk-unit block** to the trade plan — `loss_per_share` by leg (technical stop, event-gap p95, fundamental-stress), `shares_per_risk_unit = allowed_dollar_loss / max(leg losses)`, and a standalone exposure ceiling — all ARITHMETIC from existing plan levels (entry, stop, stress, event-gap). Plus a formal `entry_state` field on the decision contract (`NO_ENTRY_AT_CURRENT / WATCH_ZONE / HURDLE_CLEARING_ENTRY / WAIT_FOR_EVENT`) derived from the blockers + price-vs-entry-ladder. **No new calibration constant** (risk-unit is arithmetic; Kelly stays non-binding). Defer the full exit taxonomy (part of a later pass). **Acceptance (GOOG):** risk-unit block computes loss-per-share off the $321.74 stop / $316.94 stress / event-gap; `entry_state = WAIT_FOR_EVENT` (earnings 1d). **Files:** `trade_plan.py`, `decision_contract.py`, `render_report.py` (a Risk-units row), tests.
