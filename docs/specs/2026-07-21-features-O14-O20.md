## O18 — Event-driven coverage invalidation — **SPEC-ONLY (reclassified 2026-07-21 under the no-guess bar)**
Two findings on inspection: (1) it would **not** have caught the flagship #8 case — the $84.75B raise (Jun 3) **predated** the coverage generation (2026-07-21), so the stale "$80B" is a coverage-**authoring accuracy** error (wrong figure vs the latest filing), not a staleness-trigger miss; the real fix is a coverage figure-vs-filing check, not an event trigger. (2) The valuable half — classifying a catalyst's **event type** (financing vs M&A vs guidance) — needs a **curated pattern list**, a heuristic that must not be guessed (same blocker as O12). Only a **dates-only** core (coverage stale if a *past* reported quarter/catalyst postdates coverage generation) is truly no-guess, and it (a) is modest-value and (b) overlaps the existing `refresh_plan._event_between` earnings/dividend detection. **Deferred:** build the dates-only coverage-staleness flag when paired with the coverage figure-vs-filing accuracy check; the event-type classifier needs a user-approved pattern list.

## O19 — Risk-unit sizing output + entry-state field — **BUILD**
**Design (review-prescribed, Portfolio-OS handoff):** add a **risk-unit block** to the trade plan — `loss_per_share` by leg (technical stop, event-gap p95, fundamental-stress), `shares_per_risk_unit = allowed_dollar_loss / max(leg losses)`, and a standalone exposure ceiling — all ARITHMETIC from existing plan levels (entry, stop, stress, event-gap). ## O20 — Report polish — **SPEC-ONLY (not verifiable in this env — reclassified 2026-07-21)**
Design is sound (reportlab natively supports `setTitle/setAuthor/setSubject` + `bookmarkPage`/`addOutlineEntry`; standard-14 fonts need no embedding). **But `reportlab`/`matplotlib` are NOT installed in the dev venv** — the PDF render tests are `skipUnless`-guarded (part of the 26 skips), and reportlab Flate-compresses content streams (raw-byte metadata assertions fail without a parser). So the PDF changes **cannot be E2E-validated here**, and shipping unverified render code violates the standing real-data-E2E gate. **Build when reportlab+matplotlib are available** (the user installs them, or a CI/render env) so `setTitle`/bookmarks can be rendered and parsed back. Design ready: metadata (ticker/date/title) + 3-page outline; untruncate the page-1 fundamental-invalidation cell in `render_report`.
